Escrow Payment System

Designed and developed the backend services for Paykita, a high-availability mobile and web-based escrow payment system engineered to ensure secure and trustworthy transactions between customers and businesses. Focus was placed on building robust internal monitoring and reconciliation reports, alongside a detailed transaction history engine (utilizing MongoDB). A critical contribution involved engineering seamless and highly secure integration services with various Indonesian banks and 3rd party Payment Gateways, utilizing a Java Springboot Microservices architecture underpinned by Kafka for resilient, event-driven communication.

Core Challenges & Goals

  • • Data Visibility:
  • Provide APIs for near real-time data display using Kafka
  • • Security & Availability:
  • Ensure Payment Gateway integration with banks runs securely and is highly available.
  • • Scalability & Integrity:
  • Maintain service integrity across distributed transaction flows.

Architecture Overview

  • Core Architecture:
      Event-Driven Microservices utilizing Java Springboot. The system is composed of small, decoupled services communicating asynchronously.
  • Ingestion & Messaging:
      Kafka acts as the high-throughput message broker, handling all transactional events for secure, asynchronous communication between services.
  • Command/Data Store:
      MongoDB is utilized as the primary transactional data store, offering high flexibility and horizontal scalability for handling high-volume payment transactions.
  • API Communication:
      REST APIs are used for synchronous external communication (mobile/web requests) and for secure interaction with 3rd party bank integration services.
  • Query/Reporting:
      Dedicated Microservices and MongoDB collections are used to generate and serve internal monitoring and reporting data, optimized for fast retrieval.
