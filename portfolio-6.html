Vehicle Counting and Classification at Crossroads Using YOLOv3

Executed a final-year academic research project focused on building an automated Vehicle Counting and Classification system for crossroads using Deep Learning. The system utilized the state-of-the-art YOLOv3 (You Only Look Once) model for real-time object detection, combined with custom Image Cropping techniques to optimize detection performance in complex intersection scenarios. The project involved rigorous data annotation, training the model on custom datasets, and successfully validating the system's ability to accurately count and classify various vehicle types passing through an intersection.

Core Challenges & Goals

  • • Problem (Accuracy):
  • Traditional methods for traffic counting were manual and inaccurate. The problem was to create an automated system that could perform object detection and counting with high precision in dynamic, real-world traffic scenarios.
  • • Goal (Model Performance):
  • Train and tune the YOLOv3 model and implement image preprocessing techniques (cropping) to minimize false positives/negatives, aiming for a high detection and classification accuracy metric.
  • • Goal (Practical Output):
  • Successfully generate reliable, time-series data outputs on vehicle counts (e.g., cars, motorcycles) to support traffic flow analysis and urban planning studies.

Architecture Overview

  • Input Layer:
      Raw video streams or image sequences of a crossroad are ingested.
  • Preprocessing Layer:
      Image Cropping techniques are applied using OpenCV to focus the detection area and reduce computational overhead for the model.
  • Model Core:
      YOLOv3 (You Only Look Once) framework serves as the core deep learning model for feature extraction and bounding box generation.
  • Tracking/Counting Logic:
      Custom Python/OpenCV scripts implement logic to track detected objects across frames and increment the count once a vehicle crosses a defined virtual line.
  • Output Layer:
      Output data (vehicle counts, classification, and timestamps) is stored in local files (e.g., CSV/JSON) for analysis.
