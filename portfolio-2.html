HRIS Astra Graphia Information Technology

Engineered the backend for the HR recruitment module using .NET Microservices, focusing on an innovative AI-powered CV recommendation engine . The system leverages Azure OpenAI' s text embedding and ranking systems to automate accurate candidate-to-job matching, significantly optimizing the talent acquisition process.

Core Challenges & Goals

  • Maintain near-real-time projections for query consumers
  • The primary goal is to provide near real-time, accurate ranking projections (generated by Azure OpenAI) to HR consumers via the REST API, enabling instant filtering and selection of top applicants for job listings.

Architecture Overview

  • Core Architecture (Backend Monolith):
      A single, unified application built on .NET Core 6 (ASP.NET Core). This monolith consolidates the API layer, business logic (recruitment workflow), and data access layers
  • Business & API Layer:
      The .NET Core 6 application exposes REST API that serve data and processing functions (including interacting with the AI engine) to the React Native frontend.
  • AI Engine & Processing:
      The Azure OpenAI service is integrated to power the AI-powered CV recommendation engine. The monolithic backend handles the communication to generate text embeddings and retrieve ranking results.
  • Relational Data Store:
      SQL Server is used as the primary data store for all transactional and relational HR data (job postings, user security, candidate status).
  • High-Volume/Non-Relational Storage:
      Azure Data Table is utilized for storing large volumes of non-critical metadata or complex log data related to candidate processing.
  • Binary Storage (CVs):
      Azure Data Storage (Blob Storage) is employed to securely store the raw candidate documents (PDFs, DOCX) uploaded via the React Native interface.
