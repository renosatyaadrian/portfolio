ERP Astra Honda Motor (100K+ Users)

This project involves the implementation of a complex Event-Driven Microservices ERP system, engineered to support the operations of 100,000+ users and 500+ dealers across Sales, Service, and Finance modules for Astra Honda Motor (AHM).

As the .NET Backend Developer, my contributions included developing the financial backbone, maintaining MS SQL Server integrity, optimizing queries using GraphQL/Redis, and integrating an innovative AI-powered CV recommendation engine (Azure OpenAI).

Core Challenges & Goals

  • • Ingest Massive Volumes:
  • Ensuring the system can handle massive transaction volumes from 500+ dealers without degrading the performance of the Command Side (MS SQL Server).
  • • Near Real-Time Projections:
  • Maintaining up-to-date and low-latency read-optimized data served through GraphQL and RESTful APIs, supported by Kafka-based messaging.
  • • System Reliability:
  • Ensuring the complex distributed Microservices system is reliably verifiable through rigorous E2E Scenarios and high code coverage standards.

Architecture Overview

  • Ingestion & Command:
    • - .NET 8 Microservices act as Producers, publishing transactional Events to Kafka.
    • - Canonical writes (Commands) are executed against the MS SQL Server, guaranteeing ACID properties.
  • Processing & Query:
    • - Independent Consumer Groups (Microservices) consume these Events, execute asynchronous business logic.
    • - Updates the Query Side databases (optimized for reads).
  • Event Store:
      Apache Kafka serves as the persistent append-only log, acting as the historical source of truth for audit and event replay.

    Critical Performance Tuning

    • Kafka Stream Tuning:
      • - Carefully sizing partitions to match consumer parallelism.
      • - Utilizing batched commits and idempotent consumers.
      • - Optimizing serialization formats (e.g., Avro / Protobuf).
    • Database Tuning (MS SQL Server):

      Performing meticulous Query Plan Analysis and implementing efficient indexing on MS SQL Server for Command Side optimization.

    • Query Side Acceleration:

      Implementing Redis Caching and leveraging GraphQL as the Projection Layer to significantly reduce read load on the primary database and lower latency for user-facing APIs.
