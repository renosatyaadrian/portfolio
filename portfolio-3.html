Man Power Planning

Executed a critical full-stack migration of the Man Power Planning system from .NET 5 to .NET 8, ensuring seamless continuity for over 100 users across various Holding and Subholding companies. The project involved a complete rebuild of the application's legacy flow, creating a new, modern, and UX-friendly user interface (UI) using Blazor while simultaneously revamping the backend logic using .NET 8. Key responsibilities included guaranteeing smooth backend code compatibility, enhancing overall application performance, and collaborating closely with stakeholders to accommodate specific UI requirements and bug fixes.

Core Challenges & Goals

  • • Smooth Migration from .NET 5 to .NET 8 without Issues:
  • The primary goal was to successfully migrate the entire legacy application codebase and infrastructure dependencies from .NET 5 to the enhanced .NET 8 runtime, ensuring zero disruption to business operations.
  • • Accommodate UI Needs and Bugfix:
  • Modernize the outdated user experience by implementing a new, UX-friendly UI using Blazor. Simultaneously, ensure all legacy business flows are correctly rebuilt with robust bug fixes.
  • • Enhance Query Logic Backend:
  • Optimize critical data retrieval processes by analyzing and refactoring complex query logic within the backend to improve application responsiveness for 100+ users.

Architecture Overview

  • Core Architecture:
      Monolithic Application built using ASP.NET Core 8. The application combines the Presentation (Blazor), Business Logic, and Data Access layers into a single deployment unit.
  • Presentation Layer:
      Blazor framework used for developing the new, rich, and UX-friendly front-end interface, communicating with the backend via REST APIs.
  • Data Store:
      SQL Server serves as the relational data store for all Man Power Planning data, including organizational structures and planning metrics.
  • Caching:
      Redis is integrated as a separate caching layer, primarily used to store frequently accessed data or the results of slow-running queries, reducing the load on the SQL Server.

Critical Performance Tuning

  • • Code Refactoring:
  • Reduce redundant queries in the backend code by refactoring data access patterns and implementing efficient query structures in .NET 8.
  • • SQL Optimization:
  • Inspect performance using SQL Server Query Plans and add necessary indexes to enhance the performance of complex data retrieval and reporting queries.
  • • Caching Implementation:
  • Utilize Redis for caching data resulting from queries that are rarely updated or perform slowly, improving latency for repeated data requests from the Blazor UI.
