Geo Data Portal

Developed a core Geo Data Portal application utilizing a .NET Blazor Monolithic architecture to facilitate the recording and historical tracking of geographical activities based on precise latitude and longitude coordinates. The system was designed to handle high-volume, time-series data related to critical geographical phenomena, such as water discharge rates and volcanic activity logs.

Core Challenges & Goals

  • • Data Accommodation:
  • The primary goal was to successfully build an application capable of recording, storing, and visualizing precise latitude/longitude data and the complete historical timeline for critical geographic activities (e.g., water flow rates and volcanic activity logs).
  • • Full-Stack Delivery:
  • Successfully implement the full application logic—from the Blazor frontend to the REST API and multi-database connectivity—on the .NET 6 Monolithic platform.

Architecture Overview

  • Core Architecture:
      Monolithic Application built on .NET 6 (ASP.NET Core). All application layers are contained in a single deployment unit, serving the Blazor frontend via REST API.
  • Presentation Layer:
      Blazor framework used for developing the rich, interactive UI, providing tools for logging geo-activities and visualizing historical data.
  • Backend Framework:
      .NET 6 utilized for the unified backend logic, managing all business rules and the complex multi-database connection layer.
  • Data Persistence:
      Multi-Database Strategy (MySQL, MS SQL Server, PostgreSQL) implemented to potentially segregate data based on performance needs (e.g., PostgreSQL for spatial data).
  • API Communication:
      REST APIs are exposed by the .NET 6 monolith to manage synchronous data transactions (logging new activities) and serving complex historical data queries to the Blazor frontend.
